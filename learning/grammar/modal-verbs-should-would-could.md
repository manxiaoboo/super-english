## 核心思维

should/would/could 不是"应该/会/可以"的机械对应。它们本质上是"软化器"——把直白的陈述变成建议、把命令变成请求、把断言变成委婉表达。三者的共同DNA是"拉开说话者与断言的距离"，给对方留空间。should = 建议而非命令；would = 有条件/委婉/过去习惯；could = 委婉请求/过去能力/低把握推测。

## 用法矩阵

| | should | would | could |
|---|---|---|---|
| 建议 | You should go | — | You could try（弱建议） |
| 委婉请求 | — | Would you... | Could you... |
| 过去习惯 | — | would do | — |
| 过去能力 | — | — | could do |
| 推测 | 高把握(80%) | 会(90%) | 可能(30%) |
| 过去未发生 | should have done | would have done | could have done |

## 过去"本应/本会/本可"三兄弟

| | 含义 | 潜台词 |
|---|---|---|
| should have done | 本应做没做 | 后悔/批评 |
| would have done | 本会做（条件不满足） | 惋惜/假设 |
| could have done | 本可以做没做 | 遗憾/轻微抱怨 |

## 锚定例句

1. You **should have told** me you were coming! I prepared nothing!  
   （你本应该告诉我你要来！我什么都没准备！——后悔/不满）

2. When I was a kid, I **would** play outside every day.  
   （小时候我每天都会在外面玩。——过去习惯）

3. I **would** appreciate it if you **could** take a look at this code.  
   （如果你能帮我看一下这段代码，我会非常感激。——双重委婉）

## 最小对比

| | can | could |
|---|---|---|
| 功能 | 现在能力 + 随意请求 | 过去能力 + 委婉请求 + 低把握推测 |
| 请求对比 | Can you help?（直白） | Could you help?（礼貌，留退路） |

| | must | should |
|---|---|---|
| 功能 | 强制义务 | 建议 |
| You ___ go. | 你必须去（没商量） | 你应该去（建议） |

| | will | would |
|---|---|---|
| 功能 | 确定将要做 | 条件满足了会/过去习惯/委婉 |
| I ___ help. | 我会帮（确定） | 我会帮（但需条件，或不那么确定） |

## 复习问题

1. **辨识**：Could you open the window? / When I was young, I would read for hours. / I should have called her. / She could play the piano at age 5.
2. **填空**：You ______ (tell) me you were coming! I prepared nothing!
3. **造句**：如果你能帮我看看这段代码，我会非常感激。（would appreciate + could）

## 常见陷阱

- could 在"过去时间"语境里是"过去能力"，不是"请求/建议"
- should have done（本应做）和 must have done（推测肯定做了）完全不是一回事
- would 的过去习惯和 used to 意思相近但语法不同（would 不用于状态动词）
